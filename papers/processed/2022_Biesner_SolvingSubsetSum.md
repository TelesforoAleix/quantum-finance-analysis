## Abstract summary
The paper formulates the problem of detecting sum relationships in financial tables as instances of the subset sum problem and recasts this as a quadratic unconstrained binary optimization (QUBO) task. It then shows how Hopfield networks with gradient descent dynamics can be used to solve these QUBOs efficiently, and evaluates the approach on both synthetic subset sum instances and real financial reporting data. The authors discuss the method's reliability and outline future directions involving specialized hardware and quantum algorithms for auditing applications.
## Methodology
This TechRxiv preprint presents an empirical optimization study, explicitly flagged as non-peer-reviewed, focused on solving subset sum problems arising in auditing and financial table consistency checking. The authors formulate subset sum as a QUBO by minimizing the squared deviation between a target value and the sum of a selected subset, then convert the QUBO into an equivalent Ising-form objective over bipolar variables. They solve this objective using gradient-descent-like updates in Hopfield networks, where the QUBO coefficients are mapped to network weights and thresholds. The proposed solver repeatedly initializes the Hopfield network with random states, performs asynchronous single-neuron updates based on the energy gradient until convergence, and checks whether the minimum energy condition corresponding to an exact subset sum solution is reached. The method is evaluated on both synthetic subset sum instances and real financial-report data parsed from Deutsche Bank quarterly reports. Synthetic data are generated by uniformly sampling integers within specified ranges, selecting a random subset, and defining the target as that subset's sum. Real-data experiments treat table columns as sets of values and solve independent subset sum instances corresponding to known sums in the tables. Performance is assessed primarily by whether a correct solution is found within a maximum number of random restarts, along with mean runtime and mean number of runs to solution across problem configurations.

**Algorithms used:** QUBO, Ising model, Hopfield network gradient descent

**Experimental setup:** Experiments were run using the Hopfield-network-based subset sum solver on both artificial and real financial data. The algorithm used up to 1e+8 random initializations, processed in batches of 1e+4, on a single NVIDIA A100 GPU. For each restart, the network was initialized with a random bipolar state and updated until Hopfield energy convergence or until an exact zero-energy solution was found. The same solver configuration was applied to the financial data experiments.

**Dataset:** Two datasets were used: (1) artificial subset sum instances generated by uniformly sampling integers in symmetric ranges [-Xmax, Xmax] under multiple configurations of problem size n and magnitude Xmax; and (2) real financial data parsed from the Deutsche Bank Financial Data Supplement Q4 2020 spreadsheet, containing multiple sheets with quarterly financial reports from Q1 2019 to Q4 2020. The real dataset yielded 190 independent table columns grouped into assets, consincome, liabilities, and net revenues.
## Findings
- [supported] The paper reformulates the subset sum problem as a QUBO and equivalent Ising optimization problem, enabling solution via Hopfield-network energy minimization.
- [supported] A stochastic Hopfield-network gradient-descent algorithm reliably found correct subset-sum solutions on both artificial and real financial-table data within the tested setup.
- [supported] On artificial datasets, the algorithm solved all tested samples for configurations up to n=128 across Xmax values of 1e4, 1e5, and 1e6, and also solved all tested samples for n=16, 32, and 64 in the reported configurations.
- [supported] For the hardest reported artificial configuration (n=256, Xmax=1e6), the algorithm did not solve all samples within the run budget, finding solutions for 2 of 5 samples.
- [supported] On real financial data parsed from a Deutsche Bank report, the algorithm found correct solutions for all 190 subset-sum instances across four table groups under the maximum iteration budget.
- [supported] The experiments suggest that the magnitude of numbers (Xmax) affected time-to-solution more strongly than the number of values n over the tested artificial configurations.
- [supported] On real financial tables, average computation time increased with problem size, i.e. with the number of values in the table.
- [speculative] The authors argue that classical exact subset-sum algorithms are impractical for real auditing applications because of table sizes and numerical magnitudes, but this paper does not provide a direct benchmark against state-of-the-art classical solvers.
- [speculative] The paper gives an outlook that specialized hardware such as FPGAs and quantum algorithms may be useful for QUBO solving in this application domain.
- [speculative] The authors suggest the algorithm could be ready for deployment in smart auditing software in the near future.

## Experiment details
### Input
Artificial data: for each configuration, n integers were uniformly sampled between -Xmax and Xmax, k sampled integers were randomly selected, and the target T was set to their sum. Configurations included n=16, 32, 64, 128, 256; Xmax in {1e4, 1e5, 1e6}; k=4 for n=16 and k=8 otherwise; M=5 subset sum problems per configuration. Real data: parsed financial report columns from Deutsche Bank Financial Data Supplement Q4 2020 covering quarters Q1 2019 to Q4 2020; 190 independent columns total. Grouped configurations were assets (M=15, n=17, values from 1.7e7 to 1.5e14), consincome (M=49, n=31, values from -2.5e12 to 2.5e12), liabilities (M=29, n=20, values from 2.5e10 to 1.5e14), and net revenues (M=97, n=25, values from -5.3e10 to 2.5e12). Values in real tables were amounts up to multiple billions of euros, exact to one cent, with about 14 significant figures. No explicit preprocessing beyond parsing table columns is described.

### Process
1. Formulate subset sum as minimizing (x^T z - T)^2 over binary indicator vector z. 2. Convert the QUBO to an Ising-form objective over s in {-1,1}^n. 3. Construct Hopfield network parameters using W = -1/2 xx^T and theta = 1/2(xx^T 1 - 2Tx). 4. Set a maximum number of restart steps smax and initialize a counter. 5. For each restart, sample a random bipolar network state s. 6. Iteratively update the network until Hopfield energy converges: compute gradient ∇E = -Ws + theta, choose the index with maximum change (reported as i = arg min_i ∇E_i), and update the selected neuron via s_i = sign(W_i^T s - theta_i). 7. If the energy reaches the exact minimum E(s)=0, return the subset corresponding to neurons with state +1. 8. Otherwise, repeat with a new random initialization until a solution is found or the maximum number of restarts is reached. 9. For experiments, run up to 1e+8 initializations in batches of 1e+4 and record whether a correct solution is found, mean runtime, and mean number of runs until solution.

### Output
Outputs are exact subset selections solving the subset sum instances, summarized through success/failure counts, mean time to find a solution in seconds, and mean number of individual runs/restarts until a solution is found. For artificial data, the paper reports that all samples were solved for most configurations, except n=256 and Xmax=1e6 where 2 of 5 were solved within the run limit. For real financial data, all subset sum problems were solved within the maximum number of iterations. Results are compared across problem sizes n, number magnitudes Xmax, and financial table groups, but no classical algorithm baseline or quantum baseline is experimentally benchmarked.
